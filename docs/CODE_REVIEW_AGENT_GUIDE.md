## 概述

### 什么是代码审查专家代理？

代码审查专家代理是一个专门化的 AI 助手，提供：

- **深度代码分析**：不仅发现问题，更分析根本原因
- **长期改进规划**：制定分阶段的质量提升路线图
- **团队培训建议**：个性化的能力提升计划
- **最佳实践指导**：行业标准和最佳实践传授

### 与基础 `/cr` 命令的区别

| 特性 | `/cr` 命令 | 代码审查专家代理 |
|------|-----------|----------------|
| 问题发现 | ✅ | ✅ |
| 根本原因分析 | ❌ | ✅ |
| 模式识别 | ❌ | ✅ |
| 架构洞察 | ❌ | ✅ |
| 趋势分析 | ❌ | ✅ |
| 改进路线图 | ❌ | ✅ |
| 团队培训 | ❌ | ✅ |
| 长期规划 | ❌ | ✅ |

### 核心优势

1. **不仅发现问题**：理解为什么会出现，如何预防
2. **不仅修复代码**：提升团队整体能力
3. **不仅一次审查**：建立持续改进机制
4. **不仅技术建议**：战略规划和流程优化

```
## 代码审查与培训报告

### 问题 #1: SQL 注入漏洞（教学）

❌ 当前代码：
\`\`\`go
query := "SELECT * FROM users WHERE id = " + userID
rows, err := db.Query(query)
\`\`\`

🔍 为什么这是问题？
SQL 注入是一种安全漏洞，攻击者可以通过操纵输入来执行恶意 SQL 语句。
例如，如果 userID 是 `"1 OR 1=1"`，查询会返回所有用户。

✅ 正确做法：
\`\`\`go
query := "SELECT * FROM users WHERE id = ?"
rows, err := db.Query(query, userID)
\`\`\`

📚 学习要点：
1. **永远不要拼接 SQL 查询**：使用参数化查询
2. **验证所有输入**：确保输入符合预期格式
3. **使用 ORM 框架**：大多数现代 ORM 提供自动保护

🎯 实践练习：
1. 修复当前代码中的所有 SQL 注入漏洞
2. 编写一个测试来验证防护是否有效
3. 审查其他代码，寻找类似问题

📖 推荐阅读：
- OWASP SQL Injection: https://owasp.org/www-community/attacks/SQL_Injection
- The Bobby Tables: http://bobby-tables.com/
```

```
### 🔴 严重问题（P0 - 必须立即修复）

#### 1. SQL 注入漏洞（database.go:45）

**严重性**: 🔴 P0 - 必须立即修复
**类别**: 安全性 - SQL 注入
**预估修复时间**: 30 分钟

**当前代码**:
\`\`\`go
query := "SELECT * FROM users WHERE id = " + userID
rows, err := db.Query(query)
if err != nil {
    return nil, err
}
\`\`\`

**问题分析**:
直接拼接用户输入到 SQL 查询中，存在严重的 SQL 注入风险。攻击者可以通过构造特殊的输入来执行任意 SQL 语句。

**影响**:
- 数据泄露：攻击者可以获取所有用户数据
- 数据篡改：攻击者可以修改或删除数据
- 认证绕过：攻击者可能绕过登录验证

**修复方案**:
\`\`\`go
query := "SELECT * FROM users WHERE id = ?"
rows, err := db.Query(query, userID)
if err != nil {
    return fmt.Errorf("query user: %w", err)
}
defer rows.Close()
\`\`\`

**需要添加的测试**:
\`\`\`go
func TestQueryUser_SQLInjection(t *testing.T) {
    tests := []struct {
        name     string
        userID   string
        wantErr  bool
    }{
        {"normal", "123", false},
        {"injection_1", "1' OR '1'='1", true},
        {"injection_2", "1; DROP TABLE users--", true},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            _, err := db.QueryUser(tt.userID)
            if (err != nil) != tt.wantErr {
                t.Errorf("QueryUser() error = %v, wantErr %v", err, tt.wantErr)
            }
        })
    }
}
\`\`\`

- Go database/sql 最佳实践: https://go.dev/doc/database/manage-connections
```

## 最佳实践

### 1. 何时使用代码审查专家代理

✅ **最适合**：
- 重要功能开发完成时
- 架构级变更后
- 定期质量检查（月度/季度）
- 团队有新人时
- 发现系统性问题时
- 启动质量改进计划时

❌ **可能不需要**：
- 简单的一次性检查（使用 `/cr` 即可）
- 快速原型开发
- 个人学习项目
- 文档修改

### 2. 如何获得最佳效果

#### 提供充分上下文

```
❌ 不好的请求:
"审查代码"

✅ 好的请求:
"请审查 src/payment/ 目录，这是支付系统的核心代码，
特别关注安全性（处理金融交易）和性能（高并发场景）。
团队有 2 个新人，需要详细的指导。"
```

#### 明确关注点

```
✅ 明确安全优先:
"重点关注安全性，这是处理用户数据的代码"

✅ 明确性能优先:
"这是性能关键的路径，请详细分析性能问题"

✅ 明确教学目标:
"我们团队有很多新人，请以教学为导向进行审查"
```


## 故障排除

### 问题 1: 代理没有响应

**可能原因**:
- 代理配置问题
- 权限问题
- 网络问题

**解决方案**:
1. 检查代理文件是否存在：`agents/code-review-expert.md`
2. 检查文件格式是否正确（Markdown + frontmatter）
3. 查看错误日志

### 问题 2: 审查速度慢

**可能原因**:
- 代码量太大
- 选择了深度审查策略

**解决方案**:
1. 分批审查，每次审查单个模块
2. 使用快速审查策略
3. 专注于特定文件或目录

### 问题 3: 输出太长

**可能原因**:
- 选择了详细的输出格式
- 发现了大量问题

**解决方案**:
1. 明确指定输出格式（快速模式）
2. 限制审查范围
3. 聚焦于特定类型的问题

### 问题 4: 建议不符合实际

**可能原因**:
- 缺少上下文信息
- 项目有特殊约束

**解决方案**:
1. 提供更多上下文（业务需求、技术约束）
2. 明确项目的特殊要求
3. 询问代理是否可以调整建议

## FAQ

### Q1: 代码审查专家代理会访问外部服务吗？

**A**: 不会。代理完全在本地运行，不会将代码发送到任何外部服务。

### Q2: 代理支持哪些编程语言？

**A**: 代理支持 20+ 编程语言，包括 Go、Python、JavaScript、TypeScript、Java、C/C++、C#、PHP、Ruby、Rust、Swift、Kotlin 等。

### Q3: 审查结果会保存吗？

**A**: 代理会在当前目录创建跟踪文件 `.review-tracking.md` 记录发现的问题。你也可以要求代理以其他格式保存报告。

### Q4: 如何定制审查规则？

**A**: 编辑 `agents/review-config.json` 文件，可以自定义：
- 质量阈值
- 忽略的文件和模式
- 严重性级别
- 报告格式

### Q5: 代理可以集成到 CI/CD 流程中吗？

**A**: 目前代理主要用于交互式审查。如果需要 CI/CD 集成，建议使用基础的 `/cr` 命令，并配置 `review-config.json` 中的集成选项。

### Q6: 如何跟踪改进进度？

**A**: 代理会自动生成问题跟踪表。你也可以要求代理：
- 建立指标跟踪系统
- 设置定期审查提醒
- 生成进度报告

### Q7: 代理建议的优先级如何确定？

**A**: 优先级基于：
- **P0（严重）**: 安全漏洞、关键 bug
- **P1（高）**: 性能问题、资源泄漏
- **P2（中）**: 代码质量、可维护性
- **P3（低）**: 代码风格、文档

### Q8: 可以让代理只关注特定类型的问题吗？

**A**: 可以。在请求中明确说明：
```
"请只关注安全问题，忽略其他类型的问题"
"请重点关注性能问题"
```

### Q9: 代理可以帮助修复问题吗？

**A**: 可以。在审查完成后，你可以要求代理：
```
"请帮我修复这些问题"
"请提供修复的代码示例"
```

### Q10: 如何获得最佳的培训效果？

**A**:
1. 选择"培训审查"策略
2. 要求详细的解释和教学
3. 要求生成个性化的学习计划
4. 要求推荐学习资源
5. 安排定期的培训会话
